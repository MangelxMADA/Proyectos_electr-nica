On/Off Repetitivo

Plataforma: ESP32 Dev ModuleProfesor: Msc. Nestor Fabio Montoya Palacios

I. Introducción

Este proyecto consiste en hacer parpadear un LED conectado al pin GPIO 4 del ESP32, alternando entre estados de encendido y apagado con intervalos de 1 segundo. Es el punto de partida para comprender conceptos clave como la configuración de pines GPIO, el uso de funciones de tiempo (delay), la comunicación serial para depuración y el flujo básico de un programa Arduino (setup/loop).

El ESP32 es un microcontrolador de bajo costo y alto rendimiento desarrollado por Espressif Systems, con conectividad Wi-Fi y Bluetooth integrada, ideal para proyectos educativos y aplicaciones IoT profesionales.

II. Objetivo

Implementar un sistema de parpadeo automático de un LED utilizando el microcontrolador ESP32, demostrando el uso correcto de salidas digitales, la estructura de un programa Arduino y la comunicación serial como herramienta de monitoreo.

El siguiente esquema muestra las conexiones físicas del circuito. El LED rojo está conectado al pin D4 (GPIO4) del ESP32 a través de una resistencia limitadora de corriente de 220 $\Omega$. El cátodo del LED se conecta a la línea de tierra (GND) del ESP32.

Esquema de conexión del circuito On/Off Repetitivo (Fritzing)

Descripción de las Conexiones

  1. Pin D4 (GPIO4) del ESP32 se conecta al ánodo (+) del LED mediante un cable jumper.
  2. El cátodo (-) del LED se conecta a un extremo de la resistencia de 220 $\Omega$.
  3. El otro extremo de la resistencia se conecta al bus de tierra (GND) de la protoboard.
  4. El pin GND del ESP32 se conecta al bus de tierra de la protoboard.

V. Código Fuente

A continuación se presenta el código completo del proyecto con explicación detallada de cada sección:

On_Off_Repetitivo.ino// -- Configuración ---------------------------------- const int PIN_LED = 4; // Pin GPIO donde está el LED const int TIEMPO_ENCENDIDO = 1000; // Tiempo encendido (ms) const int TIEMPO_APAGADO = 1000; // Tiempo apagado (ms) // -- Setup ------------------------------------------- void setup() { Serial.begin(115200); Serial.println("=================================="); Serial.println(" Parpadeo de LED con el ESP32 "); Serial.println("=================================="); // Establecemos el pin del LED como salida pinMode(PIN_LED, OUTPUT); // Apagamos el LED al iniciar digitalWrite(PIN_LED, LOW); Serial.println("> Sistema listo. Iniciando parpadeo..."); } // -- Loop -------------------------------------------- void loop() { // Encendemos el LED digitalWrite(PIN_LED, HIGH); Serial.println("LED: ENCENDIDO"); delay(TIEMPO_ENCENDIDO); // Apagamos el LED digitalWrite(PIN_LED, LOW); Serial.println("LED: APAGADO"); delay(TIEMPO_APAGADO); }

Análisis del Código

  • Configuración: Se definen tres constantes (PIN_LED, TIEMPO_ENCENDIDO y TIEMPO_APAGADO) que parametrizan el comportamiento del circuito de forma modular.
  • setup(): Inicializa la comunicación serial a 115200 baudios, configura el pin GPIO4 como salida (OUTPUT) y asegura que el LED inicie apagado.
  • loop(): Se ejecuta infinitamente encendiendo el LED enviando 3.3V al pin, imprimiendo su estado en el Monitor Serial, y pausando la ejecución por 1 segundo usando delay(), para luego repetir el proceso apagando el LED (0V).

VI. Principio de Funcionamiento

El circuito opera bajo el principio básico de control digital de un GPIO (General Purpose Input/Output). Cuando el ESP32 escribe un valor HIGH en el pin GPIO4, este entrega aproximadamente 3.3V, permitiendo que la corriente fluya a través del LED y la resistencia. La resistencia de 220 $\Omega$ limita la corriente para evitar dañar el componente:

(1)$$I = \frac{V_{gpio} - V_{led}}{R} = \frac{3.3\text{ V} - 2.0\text{ V}}{220\,\Omega} \approx 5.9\text{ mA}$$

Este valor está dentro del rango seguro de operación tanto para el LED (máx 20mA) como para el pin del ESP32 (máx 40mA por pin).

On/Off por Teclado

Plataforma: ESP32 Dev ModuleProfesor: Msc. Nestor Fabio Montoya Palacios

I. Introducción

Este proyecto describe de manera detallada el circuito interactivo "On/Off Keyboard", que permite al usuario controlar el encendido y apagado de un LED conectado al ESP32 mediante comandos de teclado enviados a través del Monitor Serial del Arduino IDE.

A diferencia del proyecto "On/Off Repetitivo" —donde el LED parpadea automáticamente—, en este proyecto el usuario tiene el control total: al presionar la tecla a el LED se enciende, y al presionar la tecla b el LED se apaga. Esto introduce conceptos fundamentales de comunicación serial, lectura de datos de entrada y toma de decisiones condicionales en la programación de microcontroladores.

Este proyecto sirve como segundo paso en el aprendizaje de Arduino, consolidando los conceptos de GPIO y comunicación serial mientras introduce la interacción usuario-microcontrolador en tiempo real.

II. Objetivo

Implementar un sistema de control manual de un LED utilizando el microcontrolador ESP32, donde el usuario envía comandos desde el teclado del computador a través del Monitor Serial, demostrando el uso de:

  • Salidas digitales (GPIO) para controlar un actuador (LED).
  • Comunicación serial bidireccional (lectura y escritura).
  • Estructuras condicionales (if / else if) para la toma de decisiones.
  • Lectura de caracteres individuales desde el buffer serial.

El LED está conectado al pin D4 (GPIO4) del ESP32 a través de una resistencia limitadora de corriente de 220 $\Omega$.

Descripción de las Conexiones

  1. Pin D4 (GPIO4) del ESP32 se conecta al ánodo (+) del LED mediante un cable jumper.
  2. El cátodo (-) del LED se conecta a un extremo de la resistencia de 220 $\Omega$.
  3. El otro extremo de la resistencia se conecta al bus de tierra (GND) de la protoboard.
  4. El pin GND del ESP32 se conecta al bus de tierra de la protoboard.

El diagrama esquemático simplificado es: ESP32 GPIO4 → LED → R = 220 Ω → GND.

V. Código Fuente

On_Off_Keyboard.ino// -- Configuración ---------------------------------- const int PIN_LED = 4; void setup() { Serial.begin(115200); pinMode(PIN_LED, OUTPUT); digitalWrite(PIN_LED, LOW); Serial.println("Presiona 'a' para encender el LED"); Serial.println("Presiona 'b' para apagar el LED"); } void loop() { if (Serial.available() > 0) { char tecla = Serial.read(); if (tecla == 'a' || tecla == 'A') { digitalWrite(PIN_LED, HIGH); Serial.println("LED ENCENDIDO"); } else if (tecla == 'b' || tecla == 'B') { digitalWrite(PIN_LED, LOW); Serial.println("LED APAGADO"); } } }

Análisis del Código

  • Encabezado y Constantes: Se define una única constante PIN_LED = 4. Usar una constante permite cambiar fácilmente el pin sin modificar múltiples líneas de código.
  • setup(): Inicializa la comunicación serial a 115200 baudios, configura el pin como salida y asegura que inicie apagado. Además, envía un par de mensajes de instrucciones al usuario.
  • loop(): Utiliza un modelo basado en eventos. Solo actúa cuando Serial.available() > 0. La función Serial.read() lee el byte enviado, y una estructura condicional if/else if evalúa la tecla. La aceptación de mayúsculas y minúsculas ('a' y 'A') hace el programa más robusto.

VI. Funcionamiento

El proyecto se basa en la comunicación UART (Universal Asynchronous Receiver-Transmitter) entre el ESP32 y el computador a través del cable USB. El flujo de datos es el siguiente:

  1. El usuario escribe un carácter en el Monitor Serial y al enviarlo se transmite como un byte ASCII.
  2. El chip USB-UART del ESP32 lo deposita en el buffer de recepción serial.
  3. Serial.available() detecta datos pendientes y Serial.read() lee el byte.
  4. El programa evalúa el carácter y activa o desactiva el GPIO, enviando la respuesta de vuelta al computador.

La corriente del LED se calcula igual que en el proyecto anterior:

(1)$$I = \frac{V_{gpio} - V_{led}}{R} = \frac{3.3\text{ V} - 2.0\text{ V}}{220\,\Omega} \approx 5.9\text{ mA}$$

Control con Potenciómetro

Plataforma: ESP32 Dev ModuleProfesor: Msc. Nestor Fabio Montoya Palacios

I. Introducción

Este proyecto describe de manera detallada una práctica introductoria de electrónica analógica y digital con el microcontrolador ESP32. El sistema lee la posición angular de un potenciómetro mediante el conversor analógico-digital (ADC) del ESP32, convierte el valor leído en voltaje y en un ciclo de trabajo PWM, y utiliza este último para controlar el brillo de un LED de forma continua y proporcional.

El proyecto introduce conceptos fundamentales de lectura analógica con ADC de 12 bits, conversión de valores digitales a magnitudes físicas (voltaje), modulación por ancho de pulso (PWM) para el control de actuadores, y comunicación serial para la depuración y monitoreo de datos en tiempo real.

Se trata de una práctica ideal como primer acercamiento a la interacción entre sensores analógicos y actuadores controlados digitalmente, sentando las bases para proyectos más complejos.

II. Objetivo

Implementar un sistema de control analógico-digital utilizando el microcontrolador ESP32 que lea la posición de un potenciómetro y controle proporcionalmente el brillo de un LED, demostrando el uso de:

  • Lectura de señales analógicas mediante el ADC de 12 bits del ESP32 (rango 0-4095).
  • Conversión de valores digitales crudos a magnitudes físicas (voltaje en el rango 0-3.3V).
  • Modulación por ancho de pulso (PWM) con la API moderna del ESP32.
  • Mapeo lineal entre rangos numéricos distintos con la función map().
  • Protección de valores con la función constrain().
  • Comunicación serial para monitoreo en tiempo real de las variables del sistema.

V. Código Fuente

Potenciometro.ino// -- Configuración de pines -- const int pinPot = 34; const int pinLed = 4; // -- Configuración PWM -- const int pwmChannel = 0; const int pwmFreq = 5000; const int pwmRes = 8; void setup() { Serial.begin(115200); // Configuración de la API moderna PWM ledcSetup(pwmChannel, pwmFreq, pwmRes); ledcAttachPin(pinLed, pwmChannel); } void loop() { // PASO 1: Lectura ADC (0-4095) int val = analogRead(pinPot); // PASO 2: Mapeo a rango PWM (0-255) int brightness = map(val, 0, 4095, 0, 255); brightness = constrain(brightness, 0, 255); // PASO 3: Escritura PWM ledcWrite(pwmChannel, brightness); // PASO 4: Monitoreo Serial Serial.printf("ADC:%d PWM:%d\n", val, brightness); delay(100); }

Análisis del Código

  • Declaración de Pines: GPIO34 como entrada analógica y GPIO4 como salida PWM.
  • Configuración PWM: Frecuencia a 5 kHz (suficiente para evitar flicker visible) y resolución de 8 bits (256 niveles).
  • setup(): Inicializa el serial, y configura el canal PWM asociándolo al pin del LED usando ledcAttachPin().
  • loop(): Ejecuta los pasos secuenciales de lectura analógica, mapeo lineal, aplicación del PWM y monitoreo serial. El retardo previene la saturación del monitor.

VI. Funcionamiento

El potenciómetro actúa como un divisor de voltaje. Al conectar sus extremos entre 3.3V y GND, el cursor entrega un voltaje proporcional al ángulo de rotación:

(1)$$V_{\text{cursor}} = V_{CC} \times \frac{R_2}{R_1 + R_2} = V_{CC} \times \frac{\theta}{\theta_{\max}}$$

El ADC de 12 bits convierte el voltaje analógico en un valor digital (0 a 4095), ofreciendo una resolución de $\Delta V = 3.3/4095 \approx 0.806\text{ mV}$ por nivel:

(2)$$N_{ADC} = \frac{V_{in}}{V_{ref}} \times (2^{12} - 1) = \frac{V_{in}}{3.3} \times 4095$$

El valor ADC se mapea linealmente al rango PWM de 8 bits y se define un ciclo de trabajo (duty cycle) $D$ para la señal cuadrada:

(3)$$D_{PWM} = \frac{N_{ADC}}{4095} \times 255$$
(4)$$D = \frac{t_{\text{ON}}}{T} \times 100\%$$

La alta frecuencia del PWM (5kHz) hace que el encendido y apagado sea imperceptible, generando la ilusión de brillo continuo gracias a la persistencia retiniana.

VII. Conceptos Técnicos Clave

  • API PWM del ESP32: Permite configurar fácilmente la frecuencia y resolución del canal PWM para señales precisas.
  • Función map(): Realiza una interpolación lineal entre dos rangos distintos, esencial para la traducción ADC-PWM.
  • Función constrain(): Limita un valor numérico a un rango seguro, sirviendo como programación defensiva para el hardware.

Potenciómetro con Bluetooth

I. Introducción

Este proyecto describe de manera detallada una extensión del proyecto básico de potenciómetro que incorpora comunicación inalámbrica Bluetooth Classic mediante el perfil SPP (Serial Port Profile) del ESP32. El sistema lee la posición angular de un potenciómetro mediante el ADC del ESP32, controla el brillo de un LED proporcionalmente mediante PWM, y transmite los datos en tiempo real a un dispositivo móvil a través de Bluetooth, utilizando la aplicación Serial Bluetooth Terminal.

Este proyecto añade una capa de comunicación inalámbrica al control analógico-digital básico, permitiendo al usuario monitorear remotamente los valores del sensor desde un teléfono celular sin necesidad de estar conectado físicamente al computador. Se introducen los conceptos de Bluetooth Classic SPP, la librería BluetoothSerial.h del ESP32, y la transmisión dual de datos (simultáneamente por USB serial y por Bluetooth).

II. Objetivo

Implementar un sistema de control analógico-digital con monitoreo inalámbrico utilizando el ESP32 que:

  • Lea la posición de un potenciómetro mediante el ADC de 12 bits (rango 0-4095).
  • Convierta los valores digitales a voltaje (rango 0-3.3V).
  • Controle proporcionalmente el brillo de un LED mediante PWM de 8 bits.
  • Transmita los datos simultáneamente por USB Serial (al computador) y por Bluetooth Serial (al celular).
  • Permita la conexión inalámbrica desde la aplicación Serial Bluetooth Terminal usando el nombre de dispositivo "EQUIPO_MARAVILLA".

Potenciómetro con WiFi

Plataforma: ESP32 Dev ModuleProfesor: Msc. Nestor Fabio Montoya Palacios

I. Introducción

Este documento describe el proyecto "Control de LED y Monitoreo de Potenciómetro vía WiFi", que convierte el ESP32 en un servidor web local capaz de servir una página HTML completa con interfaz gráfica interactiva. El sistema permite encender y apagar un LED desde el navegador mediante un interruptor estilo industrial, y simultáneamente monitorear en tiempo real el voltaje de un potenciómetro a través de un gauge (medidor tipo reloj) generado con Google Charts.

A diferencia de los proyectos anteriores que utilizaban comunicación serial o Bluetooth, este proyecto introduce el paradigma de Internet de las Cosas (IoT): el ESP32 se conecta a una red WiFi existente, obtiene una dirección IP y sirve una aplicación web completa accesible desde cualquier dispositivo conectado a la misma red. La página web se almacena en la memoria Flash del ESP32 mediante la directiva PROGMEM y utiliza tecnologías web modernas como CSS3, JavaScript asíncrono y la API Fetch para comunicarse con el microcontrolador.

II. Objetivo

Implementar un sistema IoT con servidor web embebido en el ESP32 que:

  • Conecte el ESP32 a una red WiFi doméstica como cliente (modo estación).
  • Sirva una página web completa con HTML, CSS y JavaScript desde la memoria Flash.
  • Permita encender y apagar un LED mediante un interruptor visual en el navegador.
  • Lea continuamente el potenciómetro (GPIO34) y exponga los datos como API REST en formato JSON.
  • Muestre un gauge de voltaje en tiempo real usando Google Charts.
  • Presente los valores numéricos de ADC, voltaje y porcentaje actualizados cada 500 ms.
  • Sea accesible desde cualquier dispositivo (computador, celular, tablet) en la misma red.

Potenciómetro con ThingSpeak

Plataforma: ESP32 Dev ModuleProfesor: Msc. Nestor Fabio Montoya Palacios

I. Introducción

Este documento describe el proyecto "Monitoreo de Potenciómetro y LED vía ThingSpeak", que utiliza la plataforma de IoT en la nube ThingSpeak de MathWorks para registrar, almacenar y visualizar datos provenientes de un circuito con potenciómetro y LED controlados por el ESP32. A diferencia de los proyectos anteriores donde la visualización se realizaba localmente (monitor serial, app Bluetooth o servidor web embebido), en este proyecto los datos se envían a servidores remotos en Internet, permitiendo monitorear el comportamiento del circuito desde cualquier lugar del mundo a través de un navegador web o la aplicación móvil de ThingSpeak.

El programa lee periódicamente el voltaje del potenciómetro, alterna el estado de un LED cada 20 segundos, y transmite ambos valores al canal ThingSpeak mediante peticiones HTTP. La plataforma genera automáticamente gráficas temporales, un gauge analógico de voltaje y un indicador visual tipo lámpara para el estado del LED.

II. Objetivo del Proyecto

Implementar un sistema de telemetría IoT basado en la nube que:

  • Conecte el ESP32 a una red WiFi doméstica como cliente (modo estación).
  • Lea el valor analógico del potenciómetro (GPIO34) y lo convierta a voltaje.
  • Alterne automáticamente el estado de un LED (GPIO4) cada ciclo de envío.
  • Transmita ambos datos (voltaje y estado del LED) al servicio en la nube ThingSpeak cada 20 segundos.
  • Visualice los datos en tiempo real mediante los widgets de ThingSpeak: gráfica temporal, gauge analógico e indicador de lámpara.
  • Permita el acceso remoto a los datos desde cualquier dispositivo con conexión a Internet.

IV. Esquema de Conexión

El circuito utiliza el mismo esquema de conexión que los proyectos anteriores de potenciómetro. El potenciómetro se conecta como divisor de voltaje entre 3,3V y GND, con su cursor conectado a GPIO34 (entrada analógica del ADC1). El LED se conecta a GPIO4 a través de una resistencia de 220 $\Omega$ para limitar la corriente. La comunicación WiFi no requiere hardware adicional, ya que el ESP32 integra el módulo de radio WiFi 802.11 b/g/n en el propio chip.

Análisis del Código

  • Librerías Utilizadas: WiFi.h para conectar el ESP32 en modo estación (STA) y ThingSpeak.h para simplificar la comunicación HTTP POST con los servidores de MathWorks.
  • Configuración: Se definen el Channel ID y la Write API Key necesarios para autorizar la escritura en la base de datos de la nube.
  • Patrón No Bloqueante: La función loop() utiliza millis() en lugar de delay(), permitiendo que el procesador no quede bloqueado durante los 20 segundos de espera entre transmisiones.
  • Campos Múltiples: Se utiliza ThingSpeak.setField() para preparar los datos en memoria temporal (voltaje y estado del LED), seguidos de un único ThingSpeak.writeFields() para enviar todo en una sola petición HTTP POST.

  • Librerías de Adafruit: Adafruit_GFX.h proporciona las primitivas de dibujo (texto, formas) y Adafruit_SSD1306.h es el driver I2C de la pantalla OLED. Ambas usan Wire.h para gestionar los pines SDA/SCL.

VI. Principio de Funcionamiento

Comunicación con el Sensor DHT22 (1-Wire)

El DHT22 utiliza un protocolo propietario de un solo hilo (single-wire) para transmitir datos. El ESP32 envía una señal de inicio (pulso bajo de 1ms). El sensor responde transmitiendo 40 bits serialmente: 16 bits de humedad, 16 bits de temperatura y 8 bits de checksum. El código de la librería de Adafruit decodifica los anchos de pulso en microsegundos y valida el checksum.

Comunicación I2C con la Pantalla OLED

El protocolo I2C (Inter-Integrated Circuit) utiliza dos líneas compartidas. El ESP32 actúa como maestro y la pantalla OLED como esclavo. Los comandos y datos de los 128x64 píxeles se envían mediante tramas I2C que incluyen condición de inicio, dirección del esclavo (0x7A), control y datos.

Conversión Interna Termistor (NTC)

El sensor DHT22 contiene internamente un termistor NTC (Negative Temperature Coefficient) y un sensor capacitivo de humedad. El termistor varía su resistencia no linealmente de acuerdo a la temperatura, regido por la Ecuación de Steinhart-Hart. El circuito interno de fábrica realiza la linealización y la digitalización entregando directamente los grados Celsius.

IX. Proceso de Creación

  1. Diseño de Arquitectura de Comunicación: Se planificó un esquema centralizado donde el ESP32 recolecta mediante 1-Wire y reporta mediante UART (USB) y I2C simultáneamente.
  2. Instalación del Ecosistema Adafruit: Instalación de librerías para gráficos rasterizados y parsing digital del DHT desde el Library Manager del IDE de Arduino.
  3. Manejo de Tiempos del DHT22: La hoja de datos del DHT22 exige 2 segundos mínimos entre muestras consecutivas; por eso el bloque loop() integra un delay(2000) para evitar sobresaturación del IC interno.
  4. Diseño de Interfaz OLED: Se distribuyó el frame de 128x64 px en 3 filas lógicas. Texto pequeño (size 1) para el encabezado y texto mediano/grande (size 2) para magnitudes, asegurando legibilidad a distancia.

  • El sensor DHT22 proporciona lecturas pre-calibradas muy confiables ($\pm 0.5$ °C), con una complejidad de lectura encapsulada impecablemente por la librería de software.
  • La pantalla OLED SSD1306 permite una visualización autónoma de los datos, eliminando la dependencia del Monitor Serial del computador una vez cargado el firmware (el sistema se vuelve portable).

  • El robusto manejo de excepciones mediante comprobación de NaN garantiza que el sistema no presente falsos positivos si el sensor sufre una falla de conectividad o daño.

Comunicación con el Sensor DHT22 (1-Wire)

El DHT22 utiliza un protocolo propietario de un solo hilo (single-wire). El ESP32 envía una señal de inicio (pulso bajo de 1ms). El DHT22 responde transmitiendo 40 bits serialmente donde la duración del pulso alto diferencia los "0" y los "1".

Comunicación I2C con la Pantalla OLED

El protocolo I2C utiliza las líneas compartidas SDA (GPIO21) y SCL (GPIO22) operando con la pantalla en la dirección maestra 0x3C. La gráfica es construida sobre la RAM local para luego actualizar el hardware completo al llamar a display.display().

Renderizado Gráfico del Termómetro

El termómetro se renderiza utilizando primitivas gráficas por interpolación lineal. La matemática detalla cómo se simula visualmente la escala ascendente de temperatura invertida respecto a los ejes de la OLED, creando el "mercurio":

(1)$$y_{\text{nivel}} = y_{\text{tubo}} + \text{alto\_tubo} - \left\lfloor \frac{T - T_{\min}}{T_{\max} - T_{\min}} \times \text{alto\_tubo} \right\rfloor$$

Esto significa que a mayor temperatura, el nivel $y_{\text{nivel}}$ es menor (sube en la pantalla).

VI. Principio de Funcionamiento

Movimiento Armónico Simple (MHS)

El MHS es un movimiento periódico en el que la posición de una partícula varía sinusoidalmente con el tiempo. La ecuación general es:

(1)$$y(t) = A \sin(2\pi f t + \varphi)$$

Donde $A$ es la amplitud, $f$ es la frecuencia en Hz, y $\varphi$ es la fase inicial en radianes.

Oscilación Amortiguada

En sistemas reales, la energía se disipa por fricción. La ecuación del MHS amortiguado introduce un factor de decaimiento exponencial:

(2)$$y(t) = A \, e^{-\beta t} \sin(2\pi f t + \varphi)$$

Donde $\beta \geq 0$ es el coeficiente de amortiguamiento. Si $\beta = 0$, la oscilación es perpetua. Si $\beta > 0$, la onda decae gradualmente.

VI. Principio de Funcionamiento

El Sensor Ultrasónico HC-SR04

El HC-SR04 utiliza ondas ultrasónicas (sonido inaudible a 40 kHz) para medir distancias. El sensor emite un pulso, y el sonido viaja hasta un objeto, rebota y regresa al sensor (eco). La distancia se calcula basándose en la velocidad del sonido en el aire y el tiempo de vuelo (Time of Flight, ToF).

(1)$$d = \frac{v_s \times t}{2}$$

Con la velocidad del sonido $v_s \approx 343 \text{ m/s} = 0.0343 \text{ cm/}\mu\text{s}$, sustituimos:

(2)$$d \,[\text{cm}] = \frac{0.0343 \times t \,[\mu\text{s}]}{2} = \frac{t}{58.31} \approx \frac{t}{58.4}$$

Que es la fórmula utilizada en el código.
